
# 初步课程设计方案

本计划根据经验物理约束（empirical physics constraints）正式确定了物体选择标准，并绘制了用于混合 VLA / $\pi_0$ 评估的精确课程转换路径。

---

## 1. 结构化选择与第一阶段（Phase 1）设计

为了在捕捉高质量数据的同时高效启动初期阶段，我们从近期脚本研究中挑选了一个精确的子集。该子集在利用现有支持的同时，实现了向更复杂泛化能力的过渡。

### 第一阶段核心集合（共 10 个物体）
*我们完全依赖于 `prepare_groceries` 和 `set_table` 配置中固有支持的 10 个物体。*

| 物体 | 几何类别 | 尺寸限制 | 抓取难度 |
| :--- | :--- | :--- | :--- |
| `008_pudding_box` (布丁盒) | 长方体 | 小型 | **容易** (高度稳定，表面平整) |
| `009_gelatin_box` (明胶盒) | 长方体 | 小型 | **容易** (高度稳定，表面平整) |
| `005_tomato_soup_can` (番茄汤罐头) | 圆柱体 | 中型 | **容易** (顶视对称) |

### 选择逻辑：
1. **演示可用性（引导启动）**：此处每一个物体都已存在于 84 个预配置的 ReplicaCAD 场景实例中。这保证了我们**不需要立即手动生成新的专家轨迹（oracle trajectories）**来引导第一阶段。
2. **几何多样性**：涵盖了平面交互（盒子）、圆柱旋转对称（罐头）以及复杂的非规则拓扑结构（碗、苹果）。
3. **语义连贯性**：100% 的生态效度。每件物品都原生属于`冰箱`、`壁橱`或`厨房台面`。

## 2. 阶段转换课程规则

一旦第一阶段趋于稳定，我们需要明确的程序化指标来管理课程推广并衡量模型的塑性（plasticity）。

### 转换触发机制：*成功率阈值*
* **触发公式**：当活跃物体集的评估成功率在 100 个随机验证回合的滑动窗口中达到 **$\geq$ 85%** 时，进行阶段转换。
* **为什么不采用固定回合或时间？**：固定触发受硬件性能影响，且会武断地惩罚那些确实需要更多梯度步骤才能收敛的困难阶段。基于性能的阈值确保模型在环境切换前真正学到了表征。

### 增加速率 ($m$)
* **速率**：每次阶段转换增加 $m = 2$ 个物体。
* **选择标准（课程攀升）**：我们从剩余的 68 个 YCB 物体中开始迭代，优先选择**语义匹配度较高的困难物体**，逐渐向**异常离群（OOD）网格**过渡。
  - *第二阶段增加项*：`025_mug` (马克杯), `065-a_cups` (杯子) —— 通过带手柄的供用性（affordances）扩展厨房主题。
  - *第三阶段增加项*：`011_banana` (香蕉), `021_bleach_cleanser` (漂白清洁剂) —— 形状极难的非规则厨房/清洁用品。
  - *第四阶段增加项*：`033_spatula` (铲子), `048_hammer` (锤子) —— 代表完全 OOD 几何迁移的工具。

### 移除速率 ($n$) 与后向迁移
* **速率**：每次阶段转换移除 $n = 1$ 个物体。
* **选择标准**：我们从活跃集合中移除**最容易且稳定解决的物体**（例如：去掉 `008_pudding_box`）。
* **重新出现循环**：移除**并非永久性**。
  - *原因*：为了有力地测试**灾难性遗忘**和后向迁移（Backward Transfer）。在经过 3 个阶段后（即模型在数十万个梯度步中未见过 `pudding_box` 时），我们将其重新注入评估划分中。如果模型在先前学过的简单物体上失败，说明我们的塑性正则化（plasticity regularization）失效。

## 评估协议

### 准确率矩阵 R

在每个阶段 $t$ 之后，对截至目前见过的所有物体进行评估：

$$R_{t,i} = \text{在阶段 } t \text{ 训练后的物体 } i \text{ 的成功率}$$

这会为每个任务流生成一个下三角矩阵。由此衍生的标准持续学习指标如下：

| 指标 | 公式 | 衡量内容 |
| :--- | :--- | :--- |
| **BWT** (后向迁移) | $\frac{1}{T-1}\sum_{i=1}^{T-1}(R_{T,i} - R_{i,i})$ | 灾难性遗忘 |
| **FWT** (前向迁移) | $\frac{1}{T-1}\sum_{i=2}^{T}(R_{i-1,i} - b_i)$ | 零样本泛化能力 |
| **Intransigence** (顽固性) | $R_{i,i}^{\text{joint}} - R_{i,i}$ | 学习新物体的无能程度 |
| **Plasticity** (塑性) | $\frac{1}{T}\sum_{i=1}^{T} R_{i,i}$ | 引入时的峰值性能 |

其中 $b_i$ 是物体 $i$ 的随机策略基准，$R_{i,i}^{\text{joint}}$ 是所有物体联合训练的上限。

### 应包含的基准线（Baselines）

| 基准线 | 描述 |
| :--- | :--- |
| **Joint training** (联合训练) | 同时在所有物体上训练 —— 上限 |
| **Fine-tuning** (微调 - 无回放) | 无遗忘缓解的顺序微调 —— 下限 |
| **EWC** (弹性权重固化) | 标准正则化基准 |
| **PackNet / PNN** | 基于架构的持续学习基准 |
| **Replay buffer** (回放缓存) | 固定大小的经验回放 |
| **Your method** (本方法) | 环境演化的 VLA 课程学习 |
